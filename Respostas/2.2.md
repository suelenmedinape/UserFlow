
1. Eu escolhi o forkJoin para ganharmos performance. Como as duas requisições são independentes, não faria sentido esperar uma acabar para só então começar a outra. Com ele, as chamadas disparam juntas, em paralelo, e o código só segue adiante quando ambas retornarem com sucesso. É muito mais rápido e eficiente!
2. Para garantir a saúde do componente, eu usei o takeUntilDestroyed, que é o novo do Angular 16+. Ele limpa a inscrição automaticamente assim que o componente é destruído. Isso nos livra de toda aquela burocracia de gerenciar Subscriptions manualmente e evita vazamentos de memória (memory leaks).
3. Meu objetivo aqui foi deixar o código linear e declarativo. Ao usar os operadores do RxJS, a gente foge daquele aninhamento excessivo de funções, que torna o código difícil de ler. Além de ficar mais elegante, essa estrutura é fácil de evoluir: se amanhã uma chamada passar a depender da outra, basta 'plugar' um switchMap e a lógica continua fluida e organizada.


```typescript
private destroyRef = inject(DestroyRef);

ngOnInit(): void {
 const pessoaId = 1;
 forkJoin({
   pessoa: this.pessoaService.buscarPorId(pessoaId),
   qtd: this.pessoaService.buscarQuantidadeFamiliares(pessoaId)
 })
 .pipe(takeUntilDestroyed(this.destroyRef))
 .subscribe(({ pessoa, qtd }) => {
   this.texto = `Nome: ${pessoa.nome} | familiares: ${qtd}`;
 });
}
```
